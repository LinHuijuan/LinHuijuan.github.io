# Express

Web application framework for node.

一个基于 Node.js 封装的上层服务框架。

参考教程：http://expressjs.jser.us/guide.html

  • node的http模块创建服务与Express或Koa框架有何不同?

    • express是一个服务端框架,框架简单封装了node的http模块,express支持node原生的写法,express不仅封装好服务器，还封装了中间件、路由等特征，方便开发web服务器，换句话说express = http模块 + 中间件 + 路由。
    • http模块实现
    const http = require('http')
const server = http.createServer((req,res) => {
    res.writeHead(200, {
        'content-type': 'text/plain'
    })
    res.write('hello nodejs')
    res.end()
})
server.listen(3000)

# 一、认识Express

# 1.1 两大特性

  1. 提供大量易用接口，简化了程序的复杂度。
  2. 允许对请求处理函数进行拆分，将其重构为很多负责特定请求的小型请求处理函数，这便于模块化和后期维护。

# 1.2 核心

# 1.2.1 中间件Middleware

不采用一个巨大的 request 请求处理函数，相反我们将一系列简单的处理函数组合起来。每一个小的处理函数对应一个小任务，而这些处理函数就被称为中间件。

  • app.use
    • 往中间件队列中塞入新的中间件。
    • express中间件处理方式是线性的，next过后继续寻找下一个中间件，当然如果没有调用next()的话，就不会调用下一个函数了，调用就会被终止。

# 1.2.2 路由Routing

只有通过特定 HTTP 方法访问特定 URL 时才会触发处理函数的调用。

  • 注册路由时，app.get、app.use、app.all的区别

    • app.use(path,callback)：app.use是express用来调用中间件的方法。中间件通常不处理请求和响应，一般只处理输入数据，并将其交给队列中的下一个处理程序。

      app.use('/user')	// 只要路径以 /user 开始即可匹配（局部匹配路径）
      1

    • app.all()：是路由中指代所有的请求方式，用作路由处理，在app.use之后 可以理解为包含了app.get、app.post等的定义。（完整匹配路径）

# 1.2.3 子应用Sub-applications

将大型应用进行模块拆分。

# 1.2.4 易用的API函数

  • HTML渲染功能

const querystring=require('querystring');

module.exports={
    urlencoded(){
        return (req, res, next)=>{
            let arr=[];
            req.on('data', buffer=>{
                arr.push(buffer);
            });
            req.on('end', ()=>{

                req.body=post;
                next();
            });
        };
    }

# 1.4.4 cookie和session

  • 由于cookie的大小一般限制在4k，所以只对有需要的cookie设签名，不然cookie会变大。
const express = require('express');
const cookieParser = require('cookie-parser');

const server = express();
server.listen(8080);

server.use(cookieParser(
  'yong0lai0dang0zuo0mi0yao0de0cun0fang0zai0fu0wu0qi0'
));

server.get('/a', (req, res) => {
  // 未签名的
  console.log('cookie:', req.cookies); // cookie: { a: '1', name: 'lin' }
  // 签名的
  console.log('signed:', req.signedCookies); // signed: [Object: null prototype] { amount: '99.8' }
  res.cookie('name', 'lin');
  res.cookie('amount', 99.8, {
    httpOnly: true, // 只有服务器能修改
    // 24h = 86400s，这里的单位是毫秒
    maxAge: 14 * 86400 * 1000,
    //secure: true,         // https下才生效
    signed: true
  });

  res.send('ok');
});

const express=require('express');
const cookieSession=require('cookie-session');

const server=express();
server.listen(8080);

// 一般设为20分钟，通过控制有效期来保证安全性
server.use(cookieSession({

}));

server.get('/a', (req, res)=>{
  //console.log(req.session);
  if(!req.session['view']){
    req.session['view']=1;
  }else{
    req.session['view']++;
  }

  req.session['amount']=99.8;

  res.send(`欢迎你第${req.session['view']}到访本站，你的余额是：${req.session['amount']}`);
});

# 1.4.5 response

  • res.end()
    • 结束response - 如果服务端没有数据回传给客户端则可以直接用res.end返回，以此来结束响应过程。
  • res.send(body)
    • 如果服务端有数据可以使用res.send,可以忽略res.end，body参数可以是一个Buffer对象，一个String对象或一个Array。
  • res.render
    • 用来渲染模板文件，也可以结合模版引擎来使用。
  • res.redirect
    • 重定义到path所指定的URL，同时也可以重定向时定义好HTTP状态码（默认为302）。

    # 一、单页应用VS多页应用

    # 1.1 单页应用

    1. 页面渲染->js渲染

    2. 优点

      • 页面切换快
      1. 不用请求http。
      2. 第一次进入首页时才会请求一个HTML文件，但之后切换页面时无需请求，因为js是可以相应url变化的。
      3. js可以动态的把当前内容清除掉，再把下一个页面的内容挂载到页面上。此时路由就是前端做，而不再是后端做了。
    <template>
    // 只能有一个根标签
    <div>
        <div>Hello travel</div>
        <router-link to='/list'>list</router-link>
    </div>
</template>
    1
    2
    3
    4
    5
    6
    7
    1. 缺点
      • 首屏时间稍慢 1. 首屏既要请求html，还要请求js，返回后才展示。
      • seo差 1. 搜索引擎只识别html内容，不识别js。 2. 但是单页应用所有的页面都是靠js渲染。

    # 1.2 多页应用

    1. 页面跳转->返回HTML

    2. 优点

      • 首屏时间快（只经过一个http请求）
      • SEO效果快（SEO可以识别html内容）

    3. 缺点

      • 页面切换慢（每次切换都要发起http请求）

    # 四、转战scss

    由于某种需求，我决定把stylus改成sass。

    • sass是一种CSS预处理语言，以严格缩进语法规则来编写代码，不包括大括号和分号，语法和css类似。
    • SCSS 是 Sass 3 引入新的语法，其语法完全兼容 CSS3，并且继承了 Sass 的强大功能。也就是说，任何标准的 CSS3 样式表都是具有相同语义的有效的 SCSS 文件。它不关心缩进级别或空格，需要花括号和分号。
    • SCSS 具有与 Sass 语法完全相同的功能，弃用了一些不好的语法。

      • 转换请求和响应数据
